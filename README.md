# Wait to Elevate? Toward Adaptive Planning for Flood-Risk Management

[cite_start]This repository contains the code, data, and figures for the undergraduate honors thesis, *"Wait to Elevate? Toward Adaptive Planning for Flood-Risk Management,"* completed at the Thayer School of Engineering at Dartmouth College[cite: 1, 5, 10, 11]. 

## Project Overview

[cite_start]Current federal and state house elevation guidance typically treats elevation as an immediate, singular decision and relies on stationary flood hazard maps and static home values[cite: 17, 20, 21]. [cite_start]However, climate change and socio-economic growth are actively altering these variables[cite: 16]. 

[cite_start]This project provides a computational multi-objective decision-making framework to demonstrate how temporal flexibility (i.e., adaptive planning) may improve homeowner decision-making[cite: 22]. [cite_start]By modeling nonstationary parameters—specifically an evolving Generalized Extreme Value (GEV) flooding frequency and dynamic house appreciation rates—the code evaluates the tradeoffs between four primary objectives across a 20-year horizon[cite: 22, 23, 24, 195]:
* [cite_start]**Total Cost** [cite: 23]
* [cite_start]**Upfront Cost of Elevation** [cite: 23]
* [cite_start]**Benefit-Cost Ratio (BCR)** [cite: 23]
* [cite_start]**Lifetime Reliability** [cite: 23]

## Repository Structure

* `inputs/`: Contains the initial parameter bounds and historical data required to run the simulations (e.g., `discount.csv` and `mu_chain.csv`).
* `data/`: The destination folder for outputs generated by the objective evaluation and sensitivity analysis models. 
* `figures/`: Contains all output visualizations and diagnostic plots generated by the plotting scripts.
* [cite_start]`*.py`: The Python scripts used to run the Monte Carlo simulations, convergence testing, sensitivity analyses, and figure generation[cite: 189, 731].
